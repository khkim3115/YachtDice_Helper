# Yacht Dice — 트레이 미니 앱 (Electron)

윈도우 **시스템 트레이(우하단 알림 영역)에 상주**하는 가벼운 Electron 앱.
트레이 아이콘을 누르면 **작은 네이티브 메뉴**(슬랙/일반 앱처럼)가 뜨고, **싱글플레이**를 누르면
**트레이 위에 아주 작은 검정 팝업**이 떠서 그 안에서 **최소 기능 싱글플레이 요트다이스**를, **멀티플레이**를
누르면 같은 팝업이 **온라인 멀티플레이 로비**로 바뀝니다(웹과 동일한 Supabase 백엔드, 실시간 동기화).

게임은 자립형 [`popup.html`](popup.html)(HTML/CSS/JS 한 파일)로 구현됩니다 — **싱글플레이는 완전 오프라인**으로
동작하고, **멀티플레이/리더보드**만 네트워크가 필요합니다(없어도 싱글은 그대로 즐길 수 있습니다). 멀티플레이는
동봉한 [`vendor/supabase.js`](vendor/supabase.js)(supabase-js UMD)로 통신하며, 루트 웹 프로젝트(`../dist`)에는
의존하지 않습니다 — 루트 프로젝트와 완전히 분리(별도 `package.json`/`node_modules`)되어 영향을 주지 않습니다.

## 설치 파일 만들기

```bash
cd desktop
npm install
npm run dist
```

→ `desktop/release/YachtDice-Tray-Setup.exe` 생성(`package.json` 의 `build.nsis.artifactName` 으로
이름 고정). 실행하면 설치되고(시작 메뉴·바탕화면 바로가기 생성), 처음 실행 시 **부팅 자동 실행이 켜집니다**.
메뉴에서 끄고 켤 수 있습니다.

> **배포(GitHub Releases)** — 웹 홈의 *앱으로 받기 → 트레이 앱* 버튼은 고정 자산
> `releases/latest/download/YachtDice-Tray-Setup.exe` 를 가리킵니다([`src/ui/DownloadCards.tsx`](../src/ui/DownloadCards.tsx)).
>
> **자동 빌드** — GitHub에서 **릴리스를 publish 하면**(태그 `tray-vX.Y.Z`) [`.github/workflows/desktop-release.yml`](../.github/workflows/desktop-release.yml)
> 가 `windows-latest` 에서 `npm run dist -- --publish never` 로 설치 파일과 **자동 업데이트 메타데이터
> (`latest.yml` + `.blockmap`)** 를 빌드해 **그 릴리스에 함께 첨부**합니다(`gh release upload`).
> 따라서 보통은 *릴리스 만들기*만 하면 위 다운로드 링크가 최신본을 받고, 기존 설치본도 [자동 업데이트](#자동-업데이트)로
> 신버전을 감지합니다. (수동 빌드는 아래 절차대로 직접 만들어 업로드해도 됩니다. 자산 이름을 바꾸려면
> `artifactName` 도 함께 바꿔야 링크가 유지됩니다.)
>
> `--publish never` 인 이유: 업로드는 `gh` 스텝이 전담합니다. electron-builder 가 직접 publish 하면 기본
> 태그 규약(`v${version}`)으로 별도 릴리스를 만들어 이 저장소의 `tray-vX.Y.Z` 태그와 충돌하기 때문입니다.
> `latest.yml` 생성에 필요한 provider 정보는 `package.json` 의 `build.publish`(github) 가 제공합니다.

## 개발 실행(설치 없이)

```bash
cd desktop
npm install
npm start      # electron . — 트레이 아이콘 표시(클릭 → 메뉴 → 플레이)
```

## 트레이 메뉴 / 조작
- 트레이 아이콘 **좌클릭/우클릭** = 작은 네이티브 메뉴
- **싱글플레이** — 트레이 위 작은 팝업에서 오프라인 싱글 게임 시작
- **멀티플레이** — 같은 팝업이 온라인 로비로 전환(방 만들기/참가 → 실시간 대전)
- **Windows 시작 시 자동 실행** — 부팅 시 트레이 상주 토글
- **🔄 지금 업데이트 확인** — 즉시 업데이트 확인(라벨로 진행 상태 표시)
- **📥 업데이트 vX.Y.Z 설치** — 신버전 다운로드가 끝났을 때만 나타나며, 누르면 재시작·설치
- **종료** — 앱 완전 종료

## 자동 업데이트
설치본은 **GitHub Release 의 신버전을 자동으로 감지·다운로드**합니다([electron-updater](https://www.electron.build/auto-update)).
시작 직후 1회 + 이후 **1시간 간격**으로 확인하고, 새 버전을 찾으면 **백그라운드로 조용히 내려받습니다**.
다운로드가 끝나면 트레이 메뉴에 **📥 업데이트 설치** 항목이 생기며, **사용자가 누를 때만** 재시작·설치합니다 —
멀티 게임 중 방 상태가 갑자기 사라지지 않도록 **자동 재시작은 하지 않습니다**(Windows 토스트도 없음).

- **개발 모드(`npm start`)에선 비활성**입니다 — `app.isPackaged` 가 true 인 설치본에서만 동작합니다.
- 버전 비교는 `package.json` 의 `version`(semver) 기준이므로, 새 릴리스를 낼 땐 **버전을 반드시 올려야** 합니다.
- 동작 전제는 릴리스에 `latest.yml` 이 함께 올라가는 것입니다(위 *배포* 참고). 릴리스 태그 규약은 `tray-vX.Y.Z`.

## 게임 (popup.html)
최소 요트다이스 — 주사위 5개, 턴당 3회 굴림(보관 토글), 12 카테고리 점수 + 상단
보너스(소계 ≥63 → +35). 헬퍼·설정은 없지만(웹 버전에서 제공), **온라인 멀티플레이**와
**Top 10 리더보드**(점수 등록·조회)는 웹과 동일한 Supabase 백엔드로 지원합니다.

### 키보드 단축키 (팝업 포커스 상태)
- **Esc** — 팝업 닫기(숨김)
- **Space** — 주사위 굴리기
- **1 2 3 4 5** — 각 주사위 잠금/해제(왼→오)
- **Q W E R T Y U I O P [ ]** — 점수 12칸(원·투·쓰리·포·파이브·식스·초이스·포카드·풀하우스·스몰·라지·요트)
- 게임 종료 화면에서 **Enter** — 다시 시작(멀티는 로비로)
- (멀티 게임 중) **`** (백틱) — 우측 **상대 점수 패널** 토글

각 점수 칸에는 해당 키가 흐린 글자로 작게 표시됩니다.

## 🌐 온라인 멀티플레이
트레이 메뉴 **멀티플레이**를 누르면 팝업이 로비로 전환됩니다. 웹 버전과 **같은 Supabase 백엔드**
(방·실시간 동기화·서버 권위 주사위/점수)를 그대로 사용합니다.

- **방 만들기** — 닉네임 입력 후 생성 → 6자리 **방 코드**가 표시됩니다.
- **참가** — 닉네임 + 방 코드 입력 후 참가.
- **게임 시작** — 방장이, 2명 이상일 때 시작. 모든 인원의 팝업이 게임 화면으로 전환됩니다.
- 게임은 **서버 권위**입니다 — 주사위는 서버 RNG, 점수/턴 진행은 서버가 검증하며, 각 클라이언트는
  실시간으로 동기화된 상태를 그립니다. **내 차례**에만 굴리기/보관/기록이 가능합니다.
- **`** (백틱)으로 우측에 상대들의 **총점·진행 칸수·현재 차례**를 간단히 펼쳐 볼 수 있습니다.
- 게임이 끝나면 승자(또는 무승부)와 내 최종 점수가 표시되고, 닉네임으로 **리더보드(`mode: 'multi'`)** 등록이 가능합니다.
- **나가기**로 방을 떠납니다(✕/Esc/바깥 클릭은 *숨김*일 뿐 좌석은 유지 — 다시 열면 같은 방으로 복귀).

> 멀티플레이에는 Supabase 대시보드에서 **Anonymous sign-ins(익명 로그인)** 이 켜져 있어야 합니다(웹 버전과 동일).
>
> **2인 테스트** — 단일 인스턴스라 한 PC에서 앱 두 개를 동시에 띄울 수 없습니다. 한쪽은 트레이 앱에서 방을
> 만들고, 다른 한쪽은 배포 웹앱(`https://khkim3115.github.io/YachtDice_Helper/`)을 브라우저에서 열어
> 같은 코드로 참가하면 교차 검증할 수 있습니다.

## 🏆 리더보드 (온라인 Top 10)
웹 버전과 **같은 Supabase `leaderboard` 테이블**을 공유합니다(이 기능만 네트워크 필요).

- 헤더의 **🏆 버튼** — 온라인 **Top 10**(닉네임·점수)을 오버레이로 보여줍니다. (**Esc/Enter** 로 닫기)
- 게임 종료 화면의 **닉네임 입력 + `🏆 리더보드 등록`** — 점수를 `mode: 'desktop'` 으로 등록합니다.
  (닉네임 입력 중 **Enter** 로도 등록)
- 오프라인이면 `실패 (오프라인?)` / `불러오기 실패 (오프라인?)` 로 표시되고, 게임 플레이는 그대로 동작합니다.

## 동작 메모
- 트레이 클릭 시 **작은 네이티브 메뉴**만 표시.
- **싱글플레이/멀티플레이** 선택 시 트레이 위에 **아주 작은 팝업**(싱글 270×358 / 멀티 270×380,
  멀티에서 백틱으로 상대 패널을 펼치면 410 폭)이 떠서 그 안에서 플레이. 눈에 띄지 않게
  **어두운 단색(검정/회색)** 위주로, 헤더도 `설정` 처럼 보이게 디자인(파랑·노랑 없음).
- **팝업 바깥 클릭(포커스 상실) → 자동 숨김**(멀티 중에도 동일 — 창은 파괴되지 않고 숨겨지므로
  실시간 구독이 유지되어 다시 열면 최신 상태). 작업표시줄에는 표시되지 않음(`skipTaskbar`).
- 팝업 위치는 트레이 아이콘 기준으로 매번 우하단(작업 영역 안)에 계산해 배치.
- 완전 종료는 메뉴 **종료**로만(✕/숨김은 종료가 아님). 팝업은 미리 로드해 첫 오픈이 빠름.
- 단일 인스턴스 — 이미 실행 중이면 기존 팝업을 보여줍니다.
- 트레이·창·설치 파일 아이콘 모두 `build/icon.png`(설치본은 `resources/icon.png` 로 동봉).

## 빌드 문제 해결 (Windows)

`npm run dist` 중 `winCodeSign` 압축 해제가 **심볼릭 링크 권한** 때문에 실패할 수 있습니다
(`Cannot create symbolic link ... darwin/.../libcrypto.dylib`). 해당 심링크는 macOS용이라
Windows 빌드엔 불필요합니다. 해결 방법(택1):

- **개발자 모드 켜기** — 설정 ▸ 개인 정보 및 보안 ▸ 개발자용 ▸ 개발자 모드 ON (가장 간단).
- 관리자 권한 터미널에서 `npm run dist` 실행.
- 캐시 시드(개발자 모드/관리자 없이): `%LOCALAPPDATA%\electron-builder\Cache\winCodeSign\`
  아래의 (signtool.exe 가 들어있는) 임시 숫자 폴더를 `winCodeSign-2.6.0` 로 복사한 뒤 다시 빌드.

설치 파일은 코드 서명이 없어 첫 실행 시 SmartScreen 경고가 뜰 수 있습니다(추가 정보 ▸ 실행).

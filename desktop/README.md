# Yacht Dice — 트레이 미니 앱 (Electron)

윈도우 **시스템 트레이(우하단 알림 영역)에 상주**하는 가벼운 Electron 앱.
트레이 아이콘을 누르면 **작은 네이티브 메뉴**(슬랙/일반 앱처럼)가 뜨고, **플레이**를 누르면
**트레이 위에 아주 작은 검정 팝업**이 떠서 그 안에서 **최소 기능 싱글플레이 요트다이스**를 합니다.

게임은 자립형 [`popup.html`](popup.html)(HTML/CSS/JS 한 파일)로 구현되어 외부 의존성이 전혀
없습니다 — 루트 웹 프로젝트(`../dist`)나 네트워크 없이 **게임 플레이는 완전 오프라인**으로 동작합니다
(아래 **🏆 온라인 리더보드** 만 네트워크가 필요하며, 없어도 게임은 그대로 즐길 수 있습니다).
루트 프로젝트와도 완전히 분리(별도 `package.json`/`node_modules`)되어 영향을 주지 않습니다.

## 설치 파일 만들기

```bash
cd desktop
npm install
npm run dist
```

→ `desktop/release/YachtDice-Tray-Setup.exe` 생성(`package.json` 의 `build.nsis.artifactName` 으로
이름 고정). 실행하면 설치되고(시작 메뉴·바탕화면 바로가기 생성), 처음 실행 시 **부팅 자동 실행이 켜집니다**.
메뉴에서 끄고 켤 수 있습니다.

> **배포(GitHub Releases)** — 웹 홈의 *앱으로 받기 → 트레이 앱* 버튼은 고정 자산
> `releases/latest/download/YachtDice-Tray-Setup.exe` 를 가리킵니다([`src/ui/DownloadCards.tsx`](../src/ui/DownloadCards.tsx)).
> 빌드 산출물이 이미 그 이름이므로 **릴리스에 그대로 업로드**하면 링크가 항상 최신본을 받습니다.
> (자산 이름을 바꾸려면 `artifactName` 도 함께 바꿔야 링크가 유지됩니다.)

## 개발 실행(설치 없이)

```bash
cd desktop
npm install
npm start      # electron . — 트레이 아이콘 표시(클릭 → 메뉴 → 플레이)
```

## 트레이 메뉴 / 조작
- 트레이 아이콘 **좌클릭/우클릭** = 작은 네이티브 메뉴
- **플레이** — 트레이 위 작은 팝업에서 게임 시작
- **Windows 시작 시 자동 실행** — 부팅 시 트레이 상주 토글
- **종료** — 앱 완전 종료

## 게임 (popup.html)
최소 싱글플레이 요트다이스 — 주사위 5개, 턴당 3회 굴림(보관 토글), 12 카테고리 점수 + 상단
보너스(소계 ≥63 → +35). 헬퍼·로컬 멀티플레이·설정은 없지만(웹 버전에서 제공),
**온라인 Top 10 리더보드**(점수 등록·조회)는 웹과 동일한 Supabase 백엔드로 지원합니다.

### 키보드 단축키 (팝업 포커스 상태)
- **Esc** — 팝업 닫기(숨김)
- **Space** — 주사위 굴리기
- **1 2 3 4 5** — 각 주사위 잠금/해제(왼→오)
- **Q W E R T Y U I O P [ ]** — 점수 12칸(원·투·쓰리·포·파이브·식스·초이스·포카드·풀하우스·스몰·라지·야추)
- 게임 종료 화면에서 **Enter** — 다시 시작

각 점수 칸에는 해당 키가 흐린 글자로 작게 표시됩니다.

## 🏆 리더보드 (온라인 Top 10)
웹 버전과 **같은 Supabase `leaderboard` 테이블**을 공유합니다(이 기능만 네트워크 필요).

- 헤더의 **🏆 버튼** — 온라인 **Top 10**(닉네임·점수)을 오버레이로 보여줍니다. (**Esc/Enter** 로 닫기)
- 게임 종료 화면의 **닉네임 입력 + `🏆 리더보드 등록`** — 점수를 `mode: 'desktop'` 으로 등록합니다.
  (닉네임 입력 중 **Enter** 로도 등록)
- 오프라인이면 `실패 (오프라인?)` / `불러오기 실패 (오프라인?)` 로 표시되고, 게임 플레이는 그대로 동작합니다.

## 동작 메모
- 트레이 클릭 시 **작은 네이티브 메뉴**만 표시.
- **플레이** 선택 시 트레이 위에 **아주 작은 팝업(270×358)** 이 떠서 그 안에서 플레이. 눈에
  띄지 않게 **어두운 단색(검정/회색)** 위주로, 헤더도 `설정` 처럼 보이게 디자인(파랑·노랑 없음).
- **팝업 바깥 클릭(포커스 상실) → 자동 숨김.** 작업표시줄에는 표시되지 않음(`skipTaskbar`).
- 팝업 위치는 트레이 아이콘 기준으로 매번 우하단(작업 영역 안)에 계산해 배치.
- 완전 종료는 메뉴 **종료**로만(✕/숨김은 종료가 아님). 팝업은 미리 로드해 첫 오픈이 빠름.
- 단일 인스턴스 — 이미 실행 중이면 기존 팝업을 보여줍니다.
- 트레이·창·설치 파일 아이콘 모두 `build/icon.png`(설치본은 `resources/icon.png` 로 동봉).

## 빌드 문제 해결 (Windows)

`npm run dist` 중 `winCodeSign` 압축 해제가 **심볼릭 링크 권한** 때문에 실패할 수 있습니다
(`Cannot create symbolic link ... darwin/.../libcrypto.dylib`). 해당 심링크는 macOS용이라
Windows 빌드엔 불필요합니다. 해결 방법(택1):

- **개발자 모드 켜기** — 설정 ▸ 개인 정보 및 보안 ▸ 개발자용 ▸ 개발자 모드 ON (가장 간단).
- 관리자 권한 터미널에서 `npm run dist` 실행.
- 캐시 시드(개발자 모드/관리자 없이): `%LOCALAPPDATA%\electron-builder\Cache\winCodeSign\`
  아래의 (signtool.exe 가 들어있는) 임시 숫자 폴더를 `winCodeSign-2.6.0` 로 복사한 뒤 다시 빌드.

설치 파일은 코드 서명이 없어 첫 실행 시 SmartScreen 경고가 뜰 수 있습니다(추가 정보 ▸ 실행).
